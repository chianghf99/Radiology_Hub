# Project Rules

## 1. Schedule Parsing Rules (班表讀取與解析規則)
- **醫師休假資訊來源**：醫師休假請統一參考 **NI 班表文件最下方的說明/註記**（例如：`休假：黃俊肇（7/17–7/21）`），**切勿**參考 EVT 班表內部的註記。
- **門住急 MRI 週次對應**：MRI 班表的 `W1` ~ `W5` 欄位固定對應 **「週一至週五」**，而非日曆上的第幾週。
  - `W1` = 週一
  - `W2` = 週二
  - `W3` = 週三
  - `W4` = 週四
  - `W5` = 週五
- **代班資料 (covers) 填寫**：休假期間每一天的代班對應，不論是跨天或單日，皆須在資料庫的 `covers` 中以日期為 Key 逐日完整列出，不可遺漏休假的最後一日。
- **分區/分任務代班 (split covers)**：若某天某位醫師的代班人因台北/淡水院區或任務不同而有分拆（例如：黃俊肇請假，門住 CT 台北給謝棖智，淡水給姜信帆），請在 `covers` 中將代班人設為物件，並指定 `tp`、`ds` 與 `default` 代班人。
  - 範例：`'7/17': { '黃俊肇': { tp: '謝棖智', ds: '姜信帆', default: '姜信帆' } }`
  - 系統會自動解析：台北院區的 CT 任務由 `tp` 代班，淡水院區的由 `ds` 代班，一般其他任務則由 `default` 醫師代班。
- **門住 CT 獨立代班 (ct cover)**：若某天某位醫師的「門住 CT」由特定的人代班，但其他常規任務由另一人代班（例如：6/4 魏士揚請假，門住 CT 由黃俊肇代，其餘一般工作由謝棖智代），請在 `covers` 中指定 `ct` 欄位。
  - 範例：`'6/4': { '魏士揚': { ct: '黃俊肇', default: '謝棖智' } }`
  - 系統會自動解析：門住 CT 任務由 `ct` 代班人負責，其他一般任務由 `default` 負責。

## 2. Medical Domain Knowledge (醫療領域專業術語)
- **PF (Plain Film)**：為「一般 X 光片」（常規 X 光診斷報告）的縮寫。此為一項**完全獨立**的工作項目，切勿與 CT（電腦斷層）或 MR/MRI（磁振造影）等其他影像報告工作混淆。班表備註中的「PF 代班」專指 X 光片的代班工作。
