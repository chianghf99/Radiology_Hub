# Project Rules

## 1. Schedule Parsing Rules (班表讀取與解析規則)
- **醫師休假資訊來源**：醫師休假請統一參考 **NI 班表文件最下方的說明/註記**（例如：`休假：黃俊肇（7/17–7/21）`），**切勿**參考 EVT 班表內部的註記。
- **門住急 MRI 週次對應**：MRI 班表的 `W1` ~ `W5` 欄位固定對應 **「週一至週五」**，而非日曆上的第幾週。
  - `W1` = 週一
  - `W2` = 週二
  - `W3` = 週三
  - `W4` = 週四
  - `W5` = 週五
- **代班資料 (covers) 填寫**：休假期間每一天的代班對應，皆須在資料庫的 `covers` 中以日期為 Key 逐日完整列出，不可遺漏休假的最後一日。
- **任務獨立代班 (taskKey-based covers)**：排班時沒有「預設的代班人」，各項工作的代理醫師均是個別獨立指派的。當休假醫師有多項工作需由不同人代班時，請在 `covers` 中設定為物件，依任務 Key（`angio`, `erct`, `ct`, `mri`, `ds_mri`, `picc`）指派代理人：
  * 若任務本身需要進一步區分台北與淡水院區（如門住 CT），可在該任務鍵下以物件 `{ tp, ds }` 填寫。
  * 範例一（6/4 姜信帆的 CT 與其他工作代班拆開）：
    ```javascript
    '6/4': {
      '姜信帆': { angio: '謝棖智', erct: '謝棖智', mri: '謝棖智', picc: '謝棖智', ct: '黃俊肇' }
    }
    ```
  * 範例二（7/17 黃俊肇的 CT 再拆台北與淡水院區）：
    ```javascript
    '7/17': {
      '黃俊肇': {
        ct: { tp: '謝棖智', ds: '姜信帆' },
        angio: '姜信帆', erct: '姜信帆', mri: '姜信帆', picc: '姜信帆', ds_mri: '姜信帆'
      }
    }
    ```

## 2. Medical Domain Knowledge (醫療領域專業術語)
- **PF (Plain Film)**：為「一般 X 光片」（常規 X 光診斷報告）的縮寫。此為一項**完全獨立**的工作項目，切勿與 CT（電腦斷層）或 MR/MRI（磁振造影）等其他影像報告工作混淆。班表備註中的「PF 代班」專指 X 光片的代班工作。
